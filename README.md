# Habit Logger

This is a Habit Logger application built with C# and a SQL database.

## About the Project
I followed a video tutorial for this one, and it took me about two days to get it up and running. To be honest, it had a bit of a learning curve, but I'm happy with how it turned out.

## The Journey

### What was hard?
I spent about 30 minutes just trying to get SQL working inside VS Code and C#. It probably sounds silly, but it was new information for me. Even though I have some background in SQL and VS Code, I can see how, for a beginner, all this information can feel super overwhelming.

Also, trying to follow the tutorial while keeping everything in a single file was really tough. It got messy fast, but it taught me a valuable lesson about why separating responsibilities is so important. I’ve heard about that principle before, but I know I just need more practice to really master that structure.

### What was easy?
Since I already have some experience with SQL, that part was smooth sailing. It was nice to have something familiar to lean on while I was dealing with the new stuff in C# (like the database connection).

### What I learned
1. **Code Principles:** I revisited the KISS (*Keep It Simple, Stupid*) and DRY (*Don't Repeat Yourself*) principles. I had heard about them before, but seeing them in action was a good refresher. I know I need more practice to apply them fluently when using C#.
2. **C# & SQL Connection:** This was the biggest takeaway. I learned how to connect C# to a SQL database and understand the technologies needed to make it work properly.

It was a solid exercise. It wasn't perfect, but it definitely helped me understand the architecture behind simple apps a lot better.
